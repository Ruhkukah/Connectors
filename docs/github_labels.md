# GitHub Labels

Recommended labels for Phase 0.1 hardening:

- `protocol_gap`
  Use for missing or mismatched protocol artifacts, fields, messages, or stream semantics.
- `certification_gap`
  Use for missing certification coverage, scenario gaps, or log-format gaps.
- `broker_topology_question`
  Use for unresolved broker routing, risk-check, router-placement, or feed-origin questions.
- `alorengine_mapping_gap`
  Use for gaps between native MOEX events and current AlorEngine or DTC-facing surfaces.
